# Portfolio Redesign · Phase 01 Blueprint

## Direction
This portfolio should read like an engineering journal or architect's notebook rather than a startup landing page.

## Tone
- Warm
- Editorial
- Systems-focused
- Human
- Calm
- Premium
- Minimal

## Visual rules
- Background: `#f9f1f0`
- Primary text: `#663635`
- Accent: `#e1999f`
- Secondary accent / borders: `#deb3ad`
- Large typography
- Generous whitespace
- Soft borders
- Little to no shadow
- Smooth scrolling
- Restrained transitions

## Narrative structure
1. Hero
2. Chapter 1 — Who I Am
3. Chapter 2 — Experience
4. Chapter 3 — Selected Projects
5. Chapter 4 — Hackathons & Building
6. Chapter 5 — Thinking
7. Footer / direct contact

## Hero constraints
- Name centered horizontally and vertically
- Subtitle: `Software Engineer`
- Supporting line: `Backend Systems, Applied AI, Realtime Platforms, Developer Tooling`
- No startup-style call-to-action buttons

## Experience structure
Use an editorial timeline rather than resume cards.
Order: WetAcre → Forsys → Trinitum.

## Selected projects structure
Each project should read like a case study with:
- Problem
- Architecture
- Technologies
- Key Challenges
- Outcomes

Projects in scope:
- Beacon
- Atlas
- Noble
- Kairos
- Pantha

## Hackathons & Building
Position this chapter as proof of execution:
- Speed of delivery
- Technical ownership
- Complexity handled under constraints
- Repeatability of shipping

## Thinking chapter
A home for:
- Engineering principles
- Architecture notes
- Technical writing
- Systems observations

## Dev Mode
Build the interface shell first:
- top toggle between UI Mode and Dev Mode
- terminal-style overlay
- queryable content about projects, experience, stack, and decisions

Cinder integration will be added later after the main editorial experience is stable.
